User Manual

First you need to create a report to collect tweets related to a domain.

In the following screenshot, you see the report field that should be filled.




Report name should be unique. You can not create more than one report with the same name.

Tweet collection will be done based on your terms in the keywords field. But don' forget that if you enter more than one keyword tweets that contain all keywords will be retrieved.

Number of tweets field is pretty obvious ;)

English, Turkish and all are the options for tweet languages. But sentiment analysis is working better on English tweets.

Since free version of Twitter API is used, time interval is limited. It is only possible to retrieve tweets within last 7 days.

After you filled the fields click on "Collect Tweets" button and wait for all the dates that you want to analyze is listed below as in the following screenshot.




When you see all the dates that you want to analyze is listed go to the report page and start to analyze.

Let say, you created a report related to Netflix.




When you click on "Show Tweets" button, collected tweets will be displayed in a datatable with filtering panes.




When you click on "Charts" button, tweets will be analyzed and analysis results will be displayed. Keep in mind that more tweets to analyze more time it will take.

Sentiment Distribution chart shows the overall sentiment analysis for all tweets. For technical details for sentiment analysis please visit github repository.




Following two charts also show the sentiment distribution but according to per domain and per entity.




In the wordcloud, you will see the most commonly used words in the tweets. Bigger font means that word is used more frequently.




When you click on "Draw NetworkX Chart",you will be redirected the page in which the relationship between entities are displayed.

Keep in mind that the number of tweets increases, the chart gets more complex.




By using the manage bar in the upper right, you can zoom in/out different parts of chart.




The size of the edge between nodes indicates the freguency of the entities belonging the same tweet.

The size of the node indicates the freguency of the entity's occurence in the tweet.
